# Core Concepts

After learning the basics of a game engine, the next step is understanding the systems that make games interactive.

This section introduces the core mechanics used in most games.

These concepts apply to almost every game engine, including Unity and Godot.

---

# 1. Game Loops

The game loop is the system that keeps the game running.

It repeatedly updates the game state, processes player input, and renders graphics.

## What to Learn

- update loops
- frame updates
- delta time
- rendering cycles

## Resources

Game Loop Explanation  
https://gameprogrammingpatterns.com/game-loop.html

Unity Update Function  
https://docs.unity3d.com/ScriptReference/MonoBehaviour.Update.html

Godot Game Loop  
https://docs.godotengine.org/en/stable/tutorials/scripting/idle_and_physics_processing.html

# 4. Collision Detection

Collision detection allows the game to detect when objects interact.

Examples:

- player hitting a wall
- character touching a coin
- bullet hitting an enemy

## What to Learn

- colliders
- triggers
- physics bodies

## Resources

Unity Collisions  
https://docs.unity3d.com/Manual/CollidersOverview.html

Godot Physics and Collisions  
https://docs.godotengine.org/en/stable/tutorials/physics/index.html

# 6. Game State Management

Games often have different states.

Examples include:
