# Contributing to GDG Tech Learning Hub

Thank you for your interest in improving this learning repository.

This is a structured, long-term educational resource.  
To maintain quality and clarity, please follow the guidelines below.

---

## 📌 Core Principles

All contributions must:

- Be accurate and up-to-date
- Provide real educational value
- Be clearly structured
- Avoid duplication of existing resources
- Follow the established folder format

Low-quality or unstructured submissions may be rejected.

---

## 📂 Folder Structure Rule

Every track must follow this structure:

Track-Name/
│
├── README.md
├── Fundamentals/
├── Core-Concepts/
├── Practice/
├── Projects/
├── Intermediate/
├── Advanced/
└── Extra-Resources.md

Do not create random folders.

Do not rename sections without discussion.

Consistency is mandatory.

---

## 📚 Resource Standards

Acceptable resources:

- Official documentation
- Industry-recognized learning platforms
- Well-structured courses
- High-quality tutorials
- Practical project guides

Avoid:

- Outdated tutorials
- Unverified personal blogs
- Low-effort video dumps
- Promotional content
- Duplicate resources

Every added resource must include:

- A short description
- The level (Beginner / Intermediate / Advanced)
- Why it is valuable

---

## 🔄 Pull Request Process

1. Fork the repository.
2. Create a new branch.
3. Make your changes in the appropriate track and section.
4. Ensure formatting consistency.
5. Submit a Pull Request with a clear explanation of what you added or improved.
6. Wait for review from maintainers.

Large structural changes should be discussed before implementation.

---

## 🏷 Naming Conventions

Use clear and consistent naming:

Correct:
- Fundamentals
- Core-Concepts
- Projects
- Intermediate
- Advanced

Avoid:
- week1
- misc
- random
- temp
- notes

Clarity matters.

---

## 🧠 Content Philosophy

This repository prioritizes:

- Deep understanding over speed
- Structure over quantity
- Projects over passive watching
- Consistency over chaos

If a resource does not improve clarity or learning depth, do not add it.

---

Thank you for helping maintain quality and structure in this GDG initiative.
